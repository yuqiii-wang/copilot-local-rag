Key: TRD-3
Type: Story
Parent: TRD-1
Summary: Order and Trade Lifecycle Objects
Status: Done
Assignee: Jane Smith
Description:
Define the data structures for an Order placed by a user and a resulting Trade execution.

Requirements:
- Order class: ID, Side, Quantity, Price (Limit), Type (Market/Limit implied), Timestamp, Status.
- Trade class: TradeID, RefOrderID, ExecPrice, ExecQty, TradeDate.
- Ensure unique IDs (UUID).

Technical Notes:
Order status should support NEW, FILLED, PENDING, REJECTED.
